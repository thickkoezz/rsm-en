Feature Suggestions

## Completed Features ✅

  1. Cryptographic Signatures
  - [x] Add ed25519 signature verification to extrinsics
  - [x] Replace the implicit caller with an explicit signed Transaction
  - [x] Prevent replay attacks with nonce-based transaction ordering
  - See: `src/crypto.rs`, `src/transaction.rs`, `.docs/crypto.md`

  2. Events/System Logs
  - [x] Emit events from pallets (e.g., Transfer { from, to, amount })
  - [x] Build an event log that can be queried by block number
  - See: `src/events.rs`, `src/event.rs`, `.docs/events.md`

  3. Transaction Fee Mechanism
  - [x] Add a flat fee per transaction (configurable via const generic)
  - [x] Deduct fees from sender before execution (after verification)
  - [x] Emit FeePaid events and handle insufficient balance errors
  - [x] Track total fees collected for analytics
  - See: `src/fees.rs`, `src/event.rs`, `.docs/pallets.md`

  4. Persistent Storage
  - [x] Add sled embedded key-value storage
  - [x] Serialize all pallet state to disk (system, balances, events, claims, fees)
  - [x] Load state on startup and save after each block execution
  - [x] Handle serialization of complex types (AccountId arrays, &'static str claims)
  - See: `src/storage.rs`, `.docs/storage.md`

  5. Genesis Configuration
  - [x] Define a formal genesis state struct instead of manual set_balance() calls
  - [x] Provide GenesisBuilder for easy configuration construction
  - [x] Support initial balances, claims, block number, and nonces
  - [x] Integrate with Storage to apply genesis on new runtime creation
  - See: `src/genesis.rs`
